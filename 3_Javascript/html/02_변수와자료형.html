변수(Variable) 

        * 메모리에 값을 계속 바꿔가면서 저장할 수 있는 공간
        
        * 변수에 이름을 붙이고 원하는 값을 대입하면
        필요한 곳에 변수명을 호출하여 저장된 값을 사용할 수 있다.
        
        
변수명 명명 규칙

        - 의미 있는 이름을 작성해야함.
        - 대소문자 구분해야함. (test != Test)
        - 특수 문자는 _ 만 사용가능함.(단, 첫글자 제외)
        - 숫자도 사용 가능 (단, 첫 글자 제외)
        - 카멜 표기법으로 작성해야함. js는 다 카멜인가봐.

        [카멜 표기법(Camel Case)]
        문장, 여러 단어를 변수명으로 작성할 때 이어지는 
        첫번째 단어를 대문자로 작성
        ex) user name => userName

Javascript 변수의 종류와 특징

        var : 중복 선언 가능. 재 할당 가능.  
                함수 레벨 scope이다. 
        let : 중복 선언 불가능. 재 할당 가능.
             {}(블록) 레벨 scope 이다.
        const: 중복 선언 불가능. 재 할당 불가능. 
             {}(블록) 레벨 scope 이다.
       
        1순위 : const (JS는 특정 요소를 선택해서 사용하는 경우가 많기 때문에 
                        const 상수에 고정시켜둠)
        
        
        2순위 : let
        3순위 : var (사용빈도가 많이 떨어짐. 실무에서도 왠만하면 사용하지 말라고 권장 중. 변수명 중복이 가능하기 때문에, 많은 문제를 야기시키게 된다.)

자료형

자바스크립트는 변수 선언 시 별도의 자료형을 지정하지 않음.
        -> 변수에 대입되는 값(리터럴) 에 의해 자료형이 결정되어진다.

        그래서, 자바스크립트에서 자료형의 종류는?
        - string (문자열)
        - number (숫자) -> 정수/실수 모두 포함
        - boolean (논리값) -> true(참), false(거짓)
        - object (객체) -> 배열(Array), 자바스크립트 객체 {key : value, key : value} 형태임
        - function (함수)
        - undefined (정의되지 않은(선언은 됬는데, 대입이 되지 않은) 변수)

        (참고)
        - null (타입보다는 값(리터럴)의 개념) : "참조하는 것이 없음"을 의미한다.
        ex) document.getElementById("id값") 여기서 id값을 오타내면, 
            id가 일치하는 요소가 없어서 참조할 수가 없다. 그래서, null 값이 들어옴.
