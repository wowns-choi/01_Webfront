이벤트(Event)

이벤트 : 동작, 행위
        -> 브라우저에서 할 수 있는 동작을 의미한다.
        ex) click, keyup, keydown, keypress, mouseover,mouseon,mouseout,
        mouseleave,drag,change, submit ...
        
    
        *이벤트 리스너(Event Listener)
            -> 브라우저에서 이벤트가 감지될 때까지 대기하다가
            감지가 되었을 때 연결된 기능(함수)을 수행하는 것
            -> 이벤트 리스너는 "on"이 이벤트명에 붙는다. 
            ex) onclick, onchange, onsubmit, onkeyup

        *이벤트 핸들러(Event Handler)
            -> 이벤트 리스너에 연결된 기능(함수)으로
                이벤트가 감지되었을 때 수행할 내용을 작성한 함수(function)

            ex) onclick="executeFunction()" 여기서, 
            executeFunction()이라는 함수를 이벤트핸들러라고 함.

인라인 이벤트 모델(Inline)

        -HTML 요소에, 이벤트리스너="이벤트핸들러" 이렇게 쓰는 걸 말함.

        -단점: 하나의 요소에 동일한 이벤트 핸들러를 중복 작성할 수 없다.
        이전에 작성된 이벤트 핸들러를 새로운 이벤트 핸들러가 덮어쓰기함.

        -최근에는 많이 사용하지 않음. 가끔 사용됨.
        
        -인라인 이벤트 모델 작성 시 this 의 의미 :
            이벤트가 발생한 현재 요소(이벤트가 발생한 현재 태그자체)를 반환해준다.

고전 이벤트 모델

        -HTML 요소에 이벤트 관련 코드를 작성하지 않고
        JS에서 DOM 객체에 직접 이벤트 관련 코드를 작성하는 방법을 의미한다.
    
        -하나의 요소에 동일한 이벤트 핸들러를 중복 작성할 수 없다.

표준 이벤트 모델

표준 이벤트 모델(요소.addEventListener())

        -w3c(웹 표준 재정 단체)에서 
        공식적으로 지정한 이벤트 모델

            여러 이벤트 핸들러를 추가할 수 있다 !
